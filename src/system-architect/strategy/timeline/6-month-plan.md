# 6 个月系统备考方案

> **适用人群**：基础相对薄弱 / 开发经验不足 3 年 / 换行业备考 / 希望稳扎稳打  
> **每日投入**：工作日 2~3 小时，周末 3~4 小时  
> **总时长**：24 周（约 180 天）

---

## 总体时间分配

| 阶段 | 周期 | 周数 | 时间占比 | 核心任务 |
|:---|:---|:---|:---|:---|
| 一、基础搭建 | 第 1~8 周 | 8 周 | 33% | 地毯式学习教材 + 分章节练习 |
| 二、真题入门 | 第 9~14 周 | 6 周 | 25% | 近 5 年真题第一遍 + 总结 |
| 三、专项突破 | 第 15~20 周 | 6 周 | 25% | 案例深度 + 论文写作 |
| 四、模考冲刺 | 第 21~24 周 | 4 周 | 17% | 全真模拟 + 查漏补缺 |

## 第一阶段：基础搭建（第 1~8 周）

### 阶段目标
零死角通读教材，构建完整知识体系，每个章节都做练习巩固。

### 学习路径（按优先级排序）

```
软件架构设计 (核心) → 软件工程 → 系统设计 
  → 数据库 → 网络与安全 → 操作系统 → 计算机组成 
  → 法律法规 → 数学 → 英语
```

> **原则**：先拿下高权重模块（架构 + 软件工程占约 40% 分值），再补充其他。

### 第 17~18 周：论文写作突破

#### 论文写作六步法

```
1. 选题（5 分钟）     → 选最拿手的主题
2. 列提纲（10 分钟）   → 摘要 + 正文五段核心观点
3. 写正文（90 分钟）   → 按提纲展开，一气呵成
4. 检查（10 分钟）     → 补漏、修正、字数检查
5. 默写背诵（每天 20 分钟）→ 关键段落熟到顺手写出
6. 批改迭代           → 自我批改 → 重写 → 定稿
```

## 附：常见问题

**Q：6 个月计划太长，能压缩到 4~5 个月吗？**
- 可以。主要压缩第一阶段（8 周 → 5~6 周），合并部分模块
- 但基础学科（OS/计算机组成/法规）不能跳过，至少花 1 周扫一遍
- 不建议压缩专项阶段（案例 + 论文），这是提分的关键

**Q：教材太厚看不下去怎么办？**
- 第一遍不需要精读每个字——先看目录建框架，再看章节摘要
- 配合视频课程 1.5~2 倍速过知识点，再看书加深
- 善用思维导图——把一本书浓缩到 5~8 张图

**Q：需要报培训班吗？**
- 如果自制力不足或基础薄弱，可以考虑报班（希赛/黑舟等）
- 有经验的可以不报，重点刷真题 + 自己写论文
- 无论报不报班，**真题都必须自己做**，听别人讲和自己做差距巨大

**Q：什么时候开始准备论文？**
- 不建议太早（前 2 个月），因为前期知识积累不够写不出深度
- 不建议太晚（最后 2 周），时间完全不够
- **最佳时间点：第 15~18 周**（考前 6~8 周），此时知识储备和真题积累都够用
