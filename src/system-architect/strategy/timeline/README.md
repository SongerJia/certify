# 备考时间规划

> **适用考试**：软考高级系统架构设计师  
> **考试时间**：上半年 5 月下旬 / 下半年 10 月下旬  
> **当前最新考纲**：2025 版（新增云原生、AI、区块链）  
> **通过率参考**：常年 12%~15%

---

## 考试基本盘（务必先了解）

### 三科全景

| 科目 | 题型 | 题量/选择 | 满分 | 及格线 | 时长 |
|:---|:---|:---|:---|:---|:---|
| 综合知识 | 单项选择题 | 75 道 | 75 分 | 45 分 | 连考 |
| 案例分析 | 主观大题 | 5 选 3（第 1 题必做 + 后 4 选 2） | 75 分 | 45 分 | 240 分钟 |
| 论文写作 | 论述文 | 4 选 1 | 75 分 | 45 分 | 120 分钟 |

> **三科均须 ≥ 45 分，一科不过全部重来。**

### 机考特点

- 全程计算机化考试，**不得提前交卷**
- 综合知识 + 案例分析 **连考**（上午 8:30 ~ 12:30，共 240 分钟）
- 论文单独（下午 14:30 ~ 16:30，共 120 分钟）
- 熟悉打字速度和代码编辑器操作非常重要

### 2025 考纲核心变化

| 变化点 | 影响 |
|:---|:---|
| **新增云原生架构设计** | 容器化、K8s、微服务治理成为案例分析高频考点 |
| **新增区块链技术应用** | 去中心化架构、智能合约基础概念 |
| **新增 AIGC 系统集成** | AI 工程化、大模型应用架构 |
| **传统 UML 建模权重下降** | 从约 25% 降至 15%，更多考察架构设计能力 |
| **机考全面推行** | 不再有纸质答题，代码/图表需用机考系统工具 |

> 资料来源：软考办 2025 年考试大纲更新通知

## 通用备考框架

无论选择哪个方案，整个备考过程都分为 **四个阶段**：

```mermaid
graph LR
    A[基础阶段<br/>知识体系搭建] --> B[强化阶段<br/>真题精练]
    B --> C[专项突破<br/>案例+论文]
    C --> D[模考冲刺<br/>全真模拟]
```

### 各阶段核心目标

| 阶段 | 核心目标 | 关键输出 |
|:---|:---|:---|
| 基础阶段 | 通读教材，搭建知识框架 | 思维导图 + 笔记 |
| 强化阶段 | 真题刷透，建立题感 | 错题本 + 高频考点清单 |
| 专项阶段 | 案例套路 + 论文模版 | 3~5 篇完整论文 + 案例答题模板 |
| 冲刺阶段 | 全真模拟 + 查漏补缺 | 模拟成绩稳定 ≥ 50 分 |

| **10 月 24~27 日** | 考试日 | — |

## 三科备考策略速览

### 综合知识（75 道选择）

| 知识模块 | 预估分值 | 备考优先级 | 策略 |
|:---|:---|:---|:---|
| 软件架构设计 | 18~22 分 | ★★★★★ | 核心中的核心，必须拿满 |
| 软件工程 | 10~14 分 | ★★★★☆ | 需求工程、设计模式重点 |
| 系统设计（分布式/高可用） | 8~12 分 | ★★★★☆ | 结合云原生重点复习 |
| 数据库系统 | 5~8 分 | ★★★☆☆ | SQL 优化、分库分表 |
| 计算机网络与安全 | 5~8 分 | ★★★☆☆ | 协议、加密、网络安全 |
| 操作系统 | 4~6 分 | ★★★☆☆ | 进程、内存管理 |
| 计算机组成原理 | 3~5 分 | ★★☆☆☆ | 基础概念即可 |
| 法律法规与标准化 | 3~4 分 | ★★☆☆☆ | 快速记忆 |
| 数学与经济管理 | 2~3 分 | ★☆☆☆☆ | 掌握基本公式 |
| 专业英语 | 5 分 | ★★☆☆☆ | 每天读 5 分钟技术英文 |

### 案例分析（5 选 3）

**必选策略**：第 1 题通常为基础题，必须拿下；后 4 题选自己最擅长的 2 题。

| 题型方向 | 出现频率 | 关键应对 |
|:---|:---|:---|
| 软件架构设计与评估 | 几乎每次 | 掌握 ATAM、SAAM 评估方法，质量属性场景 |
| 需求工程/结构化分析 | 高频 | DFD 数据流图、用例建模 |
| 分布式系统设计 | 高频（云原生趋势下 ↑） | 微服务拆分、数据一致性、CAP/BASE |
| 数据库设计 | 中频 | ER 图、范式、SQL 优化 |
| Web 应用设计 | 中频 | 高并发、缓存、CDN |
| 嵌入式/物联网 | 低频 | 基础概念了解即可 |

### 论文（4 选 1）

**核心原则**：提前准备 3~5 篇范文，覆盖不同主题方向。

| 热门论文主题 | 热度 | 准备建议 |
|:---|:---|:---|
| 微服务架构设计与实践 | ★★★★★ | 必准备 |
| 云原生架构演进 | ★★★★★ | 2025 考纲新增方向，优先准备 |
| 系统性能优化 | ★★★★☆ | 缓存/异步/数据库优化 |
| 高可用与容灾设计 | ★★★★☆ | 多活、异地容灾 |
| 数据架构设计（大数据） | ★★★☆☆ | 数据中台、湖仓一体 |
| 系统安全架构 | ★★★☆☆ | 零信任、纵深防御 |
| 企业应用集成 | ★★★☆☆ | SOA → 微服务演进 |

```
┌─────────────────────────────────────────────┐
│  工作日（2~3 小时）                           │
│  • 30 分钟：复习前一天内容 + 错题回顾         │

```

## 备考避坑指南

1. **不要只刷选择题**——选择题刷再多，案例和论文不过一样挂
2. **论文不要临场发挥**——没有提前准备的论文大概率低分
3. **不要忽略机考练习**——打字速度、画图工具操作需要提前适应
4. **不要平均分配时间**——架构设计和软件工程占分最重，优先投入
5. **不要只看不练**——案例和论文必须动手写，只看答案无效
6. **不要等报名再开始**——报名通常在考前 2 个月，那时再开始来不及
