机考特点

  • 全程计算机化考试，不得提前交卷
  • 综合知识 + 案例分析 连考（上午 8:30 ~ 12:30，共 240 分钟）
  • 论文单独（下午 14:30 ~ 16:30，共 120 分钟）
  • 熟悉打字速度和代码编辑器操作非常重要

通用备考框架

无论选择哪个方案，整个备考过程都分为 四个阶段：

graph LR
    A[基础阶段<br/>知识体系搭建] --> B[强化阶段<br/>真题精练]
    B --> C[专项突破<br/>案例+论文]
    C --> D[模考冲刺<br/>全真模拟]

各阶段核心目标

阶段核心目标关键输出
基础阶段通读教材，搭建知识框架思维导图 + 笔记
强化阶段真题刷透，建立题感错题本 + 高频考点清单
专项阶段案例套路 + 论文模版3~5 篇完整论文 + 案例答题模板
冲刺阶段全真模拟 + 查漏补缺模拟成绩稳定 ≥ 50 分

案例分析（5 选 3）

必选策略：第 1 题通常为基础题，必须拿下；后 4 题选自己最擅长的 2 题。

题型方向出现频率关键应对
软件架构设计与评估几乎每次掌握 ATAM、SAAM 评估方法，质量属性场景
需求工程/结构化分析高频DFD 数据流图、用例建模
分布式系统设计高频（云原生趋势下 ↑）微服务拆分、数据一致性、CAP/BASE
数据库设计中频ER 图、范式、SQL 优化
Web 应用设计中频高并发、缓存、CDN
嵌入式/物联网低频基础概念了解即可

备考避坑指南

  1. 不要只刷选择题——选择题刷再多，案例和论文不过一样挂
  2. 论文不要临场发挥——没有提前准备的论文大概率低分
  3. 不要忽略机考练习——打字速度、画图工具操作需要提前适应
  4. 不要平均分配时间——架构设计和软件工程占分最重，优先投入
  5. 不要只看不练——案例和论文必须动手写，只看答案无效
  6. 不要等报名再开始——报名通常在考前 2 个月，那时再开始来不及
